## II - Features
- **Genetic algorithm**: population, fitness, selection, order crossover, and swap mutation, over integer-encoded tours.
- **Two-opt**: refines the cheapest ~10% of each generation plus the running elite.
- **Elitism + convergence detection**: the best tour always survives; a run stops early once it stagnates, hits the known optimum, or reaches the generation cap.
- **JIT-accelerated kernels**: every hot path is `numba.njit` compiled (parallel, no-GIL, cached).
- **TSPLIB instances**: loads [TSPLIB](http://comopt.ifi.uni-heidelberg.de/software/TSPLIB95/) `EUC_2D` instances and reports percent error against known optimal tour lengths.
- **Batch driver**: sweeps the GA across all 78 listed `EUC_2D` instances, 10 runs each, writing one parseable log per run.
- **Analysis**: turns run logs into plots: tours, convergence curves, edge heat maps, and aggregate error vs. computation-time scaling across instance sizes.

The analysis tools render these results into `./stdout/analysis/`: final tours, per-run and aggregate convergence, edge-usage heat maps, and error / time scaling against instance size `n`.

## IV - Quick Start
```powershell
# 1. Create and activate a virtual environment named .env.local
python -m venv .env.local
.\.env.local\Scripts\Activate.ps1

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run the full batch (78 instances x 10 runs each)
python -m src.main
```

**Note**: `python -m src.main` is a long batch job and **overwrites** existing logs in `./stdout/runs/`. For a quick sanity check, run a single small instance instead (see [Usage](#vi--usage)). The first GA call pays a ~10s one-time `numba` compile cost.

## VI - Usage
All commands run from the **repository root** with the project virtual environment active.

### Run the full batch
Runs every instance listed in `./res/info/euc-2d-instances.txt`, `number_of_runs` times each:

```powershell
python -m src.main
```

### Run a single instance
To smoke-test without overwriting committed batch results, call `RunGeneticAlgorithm` directly with a throwaway run number, then clean up the artifact:

```python
from src.utilities.instance_utility import Euclidean2D
from src.genetic_algorithm.genetic_algorithm import RunGeneticAlgorithm

RunGeneticAlgorithm(Euclidean2D('berlin52'), 999)
```

### Generate analysis plots
The `Plot` class reads logs from `./stdout/runs/` and writes images to `./stdout/analysis/` (and instance scatter plots to `./stdout/instance-plots/`):

```python
from src.utilities.plot_utility import Plot

plot = Plot()
plot.PlotEuclidean2DInstances()                # node scatter plots for every EUC_2D instance
plot.PlotTour('berlin52', run_index=0)         # final tour for one run
plot.PlotConvergence('berlin52', 0)            # percent-error vs generation for one run
plot.PlotAggregateConvergence()                # aggregate convergence across all runs
plot.PlotHeatMap('berlin52')                   # edge-usage frequency across that instance's runs
plot.PlotComputationTimeVersusNumberOfNodes()  # computation time vs n, across instances
plot.PlotPercentErrorVersusNumberOfNodes()     # percent error vs n, across instances
```

### Regenerate the instance list
`Filter` scans the instance directory and rewrites `./res/info/euc-2d-instances.txt` with only the `EUC_2D` instances:

```python
from src.utilities.filter_utility import Filter

Filter().FilterEuclidean2DInstances()
```

## VII - Configuration
GA hyperparameters are module-level constants at the top of [./src/genetic_algorithm/genetic_algorithm.py](src/genetic_algorithm/genetic_algorithm.py):

| Constant | Default | Meaning |
|----------|---------|---------|
| `population_size` | `200` | Number of tours per generation |
| `maximum_generations` | `1000` | Hard cap on generations per run |
| `selection_size` | `7` | Tournament size for parent selection |
| `order_crossover_rate` | `0.85` | Order-crossover probability |
| `swap_mutation_rate` | `0.03` | Swap-mutation probability |
| `minimum_change` | `1e-3` | Minimum fitness gain to count as an improvement |
| `convergence_generation` | `50` | Stagnant generations before early stop |
| `two_opt_percentile` | `10` | Cheapest fraction (%) of tours refined by two-opt |

Numba behavior is controlled by `NO_GIL`, `CAN_PARALLEL`, and `CAN_CACHE` in the same file. Batch driver settings live in [./src/main.py](src/main.py):

| Constant | Default | Meaning |
|----------|---------|---------|
| `instance_type` | `'euc_2d'` | Instance family to run |
| `number_of_runs` | `10` | Runs per instance |
| `starting_run` | `0` | Starting run index (offsets log filenames) |
| `exclusions` | `set()` | Instance names to skip from the list |

Plot styling toggles (fit modes, aggregate-curve display) are attributes set in `Plot.__init__` in [./src/utilities/plot_utility.py](src/utilities/plot_utility.py).

### Key entry points
- **`RunGeneticAlgorithm(instance, run_number)`**: runs one GA on one `Euclidean2D` instance, logs every generation, and returns the elite fitness.
- **`Euclidean2D(instance_name)`**: loads `./res/tsplib/tsp/<name>.tsp` and the matching optimal fitness; exposes `GetNodes()` and `GetNumberOfNodes()`.
- **`Log`**: writes the per-run log. Its format is a contract the `Plot` parsers depend on (`n:`, `computationTime:`, `percentError:`, `tour:` lines); changing it requires updating the parsers.

### Algorithm at a glance
1. Build a random initial population of tours.
2. Each generation: tournament-select parents â†’ order crossover â†’ swap mutation â†’ two-opt the cheapest `two_opt_percentile`%.
3. Carry the elite tour forward unchanged (elitism) and refine it with two-opt.
4. Stop when stagnation reaches `convergence_generation`, the optimum is hit, or `maximum_generations` is reached.

### External
- [TSPLIB95 instance library](http://comopt.ifi.uni-heidelberg.de/software/TSPLIB95/): source of the benchmark instances and the `EUC_2D` format.
