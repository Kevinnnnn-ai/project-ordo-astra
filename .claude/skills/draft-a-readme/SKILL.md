# Instructions

- Write a README to the root directory of a project.
- Try to keep most things concise, bulleted, and to the point. However, don't skip on the explanation. Deliver a README that is easy to understand and follow, even for someone who is new to the project, but digestable at the same time.
- The README should include the following sections:
  - Project Title
    - Description — It should be a one or two sentences describing the project and its purpose.
    - Badges — They should be ordered in the following way: language version coverage, listed requirement libraries, license, and build status.
  - "Ⅰ • Table of Contents"
  - "Ⅱ • Features"
  - "Ⅲ • Demonstration"
  - "Ⅳ • Quick Start"
  - "Ⅴ • Installation"
  - "Ⅵ • Usage"
  - "Ⅶ • Configuration"
  - "Ⅷ • Reference"
  - "Ⅸ • License"
  - "Ⅹ • Authors"
  - "Ⅺ • Contact"
